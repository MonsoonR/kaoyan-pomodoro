# 考研番茄钟：单账号多设备同步版设计规格

日期：2026-07-12  
状态：用户已批准，等待实施

## 目标

把现有纯前端、本地存储版升级为部署在腾讯云轻量服务器上的单账号、多设备同步 PWA。电脑、手机和平板使用同一账号，共享任务、今日计划、设置、专注记录和全局计时器；断网时继续使用，恢复网络后自动同步。

## 已批准的产品决策

- 单账号，不开放注册；用户名和密码登录。
- 默认长期登录 30 天，允许多设备同时在线。
- 网页内可修改密码；忘记密码时通过服务器 SSH 命令重置。
- 修改密码后其他设备会话失效；设置页可查看并退出设备。
- 同步版从空数据开始，不自动迁移旧版浏览器数据。
- 在线操作立即上传；首次打开、切回前台、恢复网络时同步，并约每 30 秒拉取增量更新。
- IndexedDB 保存本地副本、同步游标和待同步操作队列。
- 普通字段采用最后被服务器接受的修改；删除、完成和归档冲突进入人工确认。
- 专注记录只追加，以全局唯一操作 ID 去重。
- 服务端只允许一个活动计时器；任一设备可查看、暂停、继续、完成或退出。
- 已有计时器时，其他设备点击开始会进入当前计时器，不新建第二个。
- Docker Compose 部署，独立子域名，Caddy 自动 HTTPS。
- SQLite 持久化存储；每天自动备份，保留最近 30 天。

## 技术架构

```text
浏览器 / PWA
      │ HTTPS
      ▼
    Caddy
   ┌──┴───────────────┐
   ▼                  ▼
React 静态前端    Fastify API
                         │
                         ▼
                 SQLite + Drizzle
```

技术栈：

- 前端：React、TypeScript、Vite、PWA、IndexedDB。
- 后端：Node.js、Fastify、TypeScript、Zod。
- 数据层：SQLite、Drizzle ORM、数据库迁移。
- 认证：Argon2id 密码哈希、HttpOnly/Secure/SameSite Cookie、会话令牌哈希存储。
- 部署：Docker Compose、Caddy、Debian 13.2。

Compose 服务：

1. `caddy`：公网 80/443、TLS、静态前端和 `/api/*` 反向代理。
2. `web`：多阶段构建 React/Vite，不保存用户数据。
3. `api`：认证、同步、任务、计时器、冲突、设备和导出接口。
4. `backup`：SQLite 在线备份、压缩、完整性校验和 30 天清理。

## 核心数据

主要表：

- `users`：唯一账号和密码哈希。
- `sessions`：设备独立登录会话。
- `devices`：设备名称、浏览器、系统和最后活跃时间。
- `tasks`：长期任务。
- `daily_tasks`：每日任务和番茄进度。
- `focus_sessions`：只追加的专注记录。
- `active_timer`：全局唯一活动计时器。
- `sync_operations`：离线操作幂等去重。
- `sync_changes`：增量拉取变更日志。
- `conflicts`：需要人工处理的冲突。
- `settings`：用户设置。

可同步实体包含 `id`、`version`、`updatedAt` 和 `deletedAt`。写操作携带设备看到的基础版本；服务端通过事务和版本检查决定正常写入、自动覆盖或创建冲突。

## 离线同步

每个本地操作包含：

```text
operationId
entityType
entityId
operationType
baseVersion
payload
createdAt
```

同步流程：

1. 用户操作先写入 IndexedDB 并立即更新界面。
2. 在线时立即提交；离线时进入队列。
3. 恢复网络后按顺序重试。
4. 服务端记录 `operationId`，重复请求不会重复执行。
5. 客户端使用增量游标拉取其他设备的更新。
6. 未解决冲突不会阻塞无关实体同步。

同步状态显示为：已同步、正在同步、离线使用（含待同步数量）、同步失败。

## 冲突规则

- 标题、科目、预计番茄数、备注：按服务器接受顺序确定最终修改。
- 删除与修改：选择保留修改、确认删除或复制为新任务。
- 完成与恢复/继续编辑：选择最终完成状态。
- 归档与加入今日：人工确认最终状态。
- 专注记录：只追加，不互相覆盖；重复操作去重。

## 全局计时器

服务端保存阶段、开始时间、目标结束时间、暂停时间、状态和版本号。浏览器根据服务端校准时间显示倒计时，不会每秒请求 API。

- 创建计时器使用数据库事务保证全局唯一。
- 暂停、继续、完成和退出必须携带版本号。
- 版本过期时重新拉取状态，不重复生成记录。
- 断网期间允许本地控制；恢复后若服务端状态不兼容，进入人工冲突确认。

## 账号安全

- 首次部署通过容器命令交互式创建账号。
- 不把明文密码写入 `.env`、Compose 文件或日志。
- Argon2id 哈希密码；数据库只保存会话令牌哈希。
- Cookie 使用 HttpOnly、Secure、SameSite=Lax。
- 登录限速和失败短暂锁定。
- 修改密码使其他会话失效。
- 支持退出指定设备和退出其他所有设备。
- SSH 密码重置后所有会话失效。

## PWA

首次在线访问并登录后：

- 缓存应用程序外壳，断网时可重新打开。
- 手机可添加到主屏幕，电脑可安装为独立窗口。
- API 数据存入 IndexedDB，不缓存密码或可读取的会话令牌。
- 新版本可用时提示刷新，不强制打断正在进行的专注。

## API 模块

```text
/api/auth/*       登录、退出、当前账号、改密
/api/devices/*    设备名称与会话管理
/api/sync/*       操作上传、增量拉取、同步状态
/api/timer/*      全局计时器查询与控制
/api/conflicts/*  冲突查看与解决
/api/settings/*   设置
/api/export       完整数据导出
/api/health/*     存活与就绪检查
```

## 部署与备份

宿主机目录建议为 `/opt/kaoyan-pomodoro`，持久化 `data/`、`backups/` 和必要日志。仅 Caddy 对公网开放端口，API 和 SQLite 不直接暴露。

更新流程：更新前备份 → 构建镜像 → 数据库迁移 → 健康检查 → 切换新版本。迁移失败时不启动新 API。

备份要求：

- 每日一次 SQLite 在线备份。
- gzip 压缩和完整性检查。
- 保留最近 30 天。
- 更新前额外备份。
- 提供指定备份恢复脚本；恢复失败自动回滚当前数据库。

## 验收标准

实现完成前必须通过：

- 登录、长期会话、改密和会话失效测试。
- 任务、今日任务和设置 API 集成测试。
- 离线队列重试、幂等和增量同步测试。
- 普通字段合并及删除/完成/归档冲突测试。
- 两台设备并发开始、暂停、继续和完成计时器测试。
- 双浏览器上下文多设备同步测试。
- PWA 断网重新打开测试。
- SQLite 备份、校验和恢复测试。
- Docker Compose 全新 Debian 环境启动测试。
- HTTPS、Cookie、安全响应头、桌面与手机完整流程检查。

最终成功标准：电脑上的任务或计时器状态能在手机自动出现；离线操作不丢失，联网后安全同步；服务器重启、容器重建、升级和数据库恢复后数据仍存在。
